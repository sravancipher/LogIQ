Cloud Integrations

Connect a Cloud Provider

Service selection appends a source hint in the webhook URL for explicit parser routing.

Setup Guide

  1. Go to AWS SNS → Create Topic (Standard)
  2. Add an HTTP/S Subscription
  3. Paste the Webhook URL shown after connecting
  4. Your app auto-confirms the subscription
  5. Route AWS services (CloudWatch, CloudTrail, GuardDuty, RDS, Lambda, Security Hub, EventBridge) to SNS/EventBridge
  6. Use the generated URL as the webhook target for selected AWS events
  1. Go to Azure Monitor → Action Groups
  2. Add a Webhook action
  3. Paste the Webhook URL (with ?token=) shown after connecting
  4. Attach Action Group/Event Grid for Monitor Alerts, Activity Log, App Insights, Service Health, Defender, AKS
  5. Logs flow in automatically when alerts/events fire
  1. Create a GCP alerting notification channel or Pub/Sub + push webhook pipeline
  2. Paste the generated Webhook URL (with token and source) as the target
  3. Connect Cloud Logging, Cloud Monitoring, and Security Command Center exports
  4. Logs and incidents flow into the platform for analysis
